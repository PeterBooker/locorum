# Verifying a Locorum release

Every Locorum release ships with three layers of integrity material:

1. **`SHA256SUMS`** — checksums for every artifact, generated by goreleaser.
2. **`*.sig` + `*.cert`** — sigstore (cosign) keyless signatures bound to the
   GitHub Actions OIDC issuer. Anyone can verify these without a public key.
3. **`locorum.intoto.jsonl`** — SLSA Level 3 provenance from the
   `slsa-github-generator` reusable workflow. Verifiable with
   [`slsa-verifier`](https://github.com/slsa-framework/slsa-verifier).

You don't need all three to trust a binary, but each layer rules out a
different attack:

| Layer | Defends against |
|---|---|
| SHA256 | Bit-rot, transport corruption, swap-the-binary attacks at the mirror. |
| cosign | Forged checksums (the signature covers the file content + a Sigstore-issued cert). |
| SLSA   | "Right binary, wrong build" — proves the artifact came from this exact commit + workflow. |

## Quick verification

```bash
# 1. Download the artifact + SHA256SUMS into the same directory.
sha256sum -c SHA256SUMS --ignore-missing
```

## Sigstore (cosign) verification

Install `cosign`:

```bash
go install github.com/sigstore/cosign/v2/cmd/cosign@latest
# or: brew install cosign
```

Verify any artifact:

```bash
cosign verify-blob \
  --certificate-identity-regexp 'https://github\.com/PeterBooker/locorum/\.github/workflows/release\.yml@.+' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  --signature   locorum-X.Y.Z-linux-amd64.tar.gz.sig \
  --certificate locorum-X.Y.Z-linux-amd64.tar.gz.cert \
  locorum-X.Y.Z-linux-amd64.tar.gz
```

If the output ends with `Verified OK` you have proof that:

- The signature was produced by GitHub's OIDC identity for this workflow.
- The artifact bytes match the signed content.

A failure here means *something is wrong*: tampered binary, swapped
artifact, or someone impersonating our release process. Do not run it.

## SLSA Level 3 provenance

```bash
go install github.com/slsa-framework/slsa-verifier/v2/cli/slsa-verifier@latest

slsa-verifier verify-artifact \
  --provenance-path locorum.intoto.jsonl \
  --source-uri github.com/PeterBooker/locorum \
  --source-tag X.Y.Z \
  locorum-X.Y.Z-linux-amd64.tar.gz
```

A successful run prints `PASSED` and the verified builder identity.
This proves the artifact was built by *this* workflow at *this* commit
on a hosted GitHub-Actions runner — not, for instance, by a personal
laptop with the same key.

## Inspecting the SBOM

Two SBOM formats ship per artifact:
- `*.cdx.json` (CycloneDX 1.5)
- `*.spdx.json` (SPDX 2.3)

```bash
# Count packages in the SBOM
jq '.components | length' < locorum-X.Y.Z-linux-amd64.tar.gz.cdx.json

# Find every component with a CVE in the OSV database
osv-scanner --sbom locorum-X.Y.Z-linux-amd64.tar.gz.cdx.json
```

## Reproducible builds (Linux only)

Locorum's Linux artifacts are reproducible — building from the same tag
on the same Go toolchain and OS produces a bit-identical binary:

```bash
git clone https://github.com/PeterBooker/locorum
cd locorum
git checkout X.Y.Z
./scripts/reproducibility-check.sh
```

The script builds twice and compares byte-for-byte. See
[`docs/BUILDING.md`](BUILDING.md) for the supported reproducible build
environment.
